## Mode Behaviors

### Manual Mode
- Each phase requires human verification before proceeding
- "Pass to Next Phase" buttons advance the pipeline
- Can regenerate any individual component
- Full visibility into all intermediate outputs

### Automatic Mode
- Full pipeline runs from Research â†’ Execution
- Human approval only at final trade execution step
- Configured checkpoints run automatically
- Email/webhook notifications at each phase

## Backend API Endpoints Needed

### Research
- `GET /api/research/current` - Get current research
- `POST /api/research/generate` - Trigger research generation
- `POST /api/research/verify` - Mark research as verified

### PM Pitches
- `GET /api/pitches/current` - Get current PM pitches
- `POST /api/pitches/generate` - Trigger PM pitch generation
- `POST /api/pitches/{id}/approve` - Approve a pitch
- `POST /api/pitches/{id}/regenerate` - Regenerate a pitch

### Council
- `GET /api/council/current` - Get current council decision
- `POST /api/council/synthesize` - Trigger council synthesis
- `POST /api/council/approve` - Approve council decision

### Trades
- `GET /api/trades/pending` - Get pending trades
- `POST /api/trades/execute` - Execute approved trades
- `POST /api/trades/{id}/modify` - Modify a trade
- `POST /api/trades/{id}/reject` - Reject a trade

### Monitor
- `GET /api/positions` - Get all positions
- `GET /api/accounts` - Get account summaries
- `POST /api/checkpoint/run` - Run checkpoint now
- `GET /api/checkpoint/history` - Get checkpoint history
